New Storage Group

A Storage Group (a.k.a. NetApp Volume), as opposed to a Storage Unit (NFS Export, CIFS Share or iSCSI LUN), is to be created in the following cases:
To group Storage Units together facilitating a consistent application-wide recovery point (Snapshot)

Name is required.

Storage Group name must be minimum 3 characters long.

Storage Group name should be maximum 64 characters long.

Only Alphanumeric, spaces and dashes are allowed

Name to refer to the NetApp Volume in this portal

Code is required.

Code is must be minimum 3 characters long.

Code is should be maximum 32 characters long.

Only lowercase alphanumeric characters and underscores are allowed
First character must be alphabetical

The name of the NetApp Volume configured in the {{labels.server.serverName}}.
It is mandatory for the Storage Group Code be unique within a {{labels.server.serverName}}.
It can be useful (but not mandatory) for the Storage Group Code be unique across the entire environment. This promotes ease-of-management and allows future replication between {{labels.server.serverName}}s without naming issues.

Annotation should be maximum 256 characters long.

Only Alphanumeric, underscore, spaces and dashes are allowed

The annotation field can be referenced by CMDBs.
e.g. To facilitate tracking owners for Change Management purposes.
This field will be entered into the NetApp Volume's Comment field
